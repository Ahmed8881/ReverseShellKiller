Detect & Terminate Malicious Reverse Shells

Reverse Shell Killer is a comprehensive security tool designed to detect and automatically terminate malicious reverse shell connections on Linux systems. With dual detection methods combining pattern-based detection and AI-powered analysis, it provides enhanced security for your systems.

The Challenge of Reverse Shell Attacks

Reverse shells represent one of the most common and persistent post-exploitation techniques used by attackers after gaining initial access to a system. Unlike traditional connection methods where a client connects to a server, reverse shells initiate connections from the compromised system back to the attacker's machine, effectively bypassing inbound firewall restrictions and network security controls.

Key Challenges:

  • Detection Evasion: Attackers continuously evolve their reverse shell techniques, using obfuscation, encoding, and non-standard implementations to evade traditional signature-based detection methods.
  • False Positives: Many legitimate system operations can resemble reverse shell behavior, making reliable detection without excessive false positives extremely difficult.
  • Response Time: The window between reverse shell establishment and an attacker gaining persistent access is often measured in minutes, requiring near-immediate detection and response.
  • Detection Gaps: Existing security solutions typically rely on network-based detection or signature matching, creating blind spots for novel or modified techniques.
  • Resource Constraints: Continuous monitoring for threats must be balanced with minimal performance impact on production systems.

System Architecture

Reverse Shell Killer is built with a modular architecture consisting of several key components that work together to provide comprehensive protection against reverse shell attacks.

Core Components

Process Scanner

Heart of the detection system that scans command line arguments and network connections, uses pattern matching against known reverse shell signatures, and calls the LLM module for advanced analysis of suspicious processes.

LLM Analyzer

Leverages Google's Gemini API for sophisticated payload analysis, analyzing process commandlines to determine their nature and returning confidence scores and detailed analysis.

Email Notifier

Sends HTML-formatted email alerts on detections, including detailed information about the detected threat and providing guidance for further investigation.

Logger

Configures logging to both console and rotated log files, provides different verbosity levels, and ensures logs don't grow uncontrollably large.

Metrics Collector

Exposes Prometheus metrics, tracks detection events, false positives, and resource usage, enabling real-time monitoring and alerting.

Configuration Manager

Parses command-line arguments, handles environment variable integration, and provides a unified configuration interface for the system.

Installation Guide

System Requirements

  • Linux-based operating system
  • Python 3.13 or higher
  • Internet connection (for LLM-based analysis)
  • SMTP access (for email notifications)

Installation Methods

Method 1: Install from Source

git clone https://github.com/youruser/reverse-shell-killer.git
cd reverse-shell-killer
pip install -r requirements.txt
python reverse_shell_killer.py

Method 2: Using uv (Recommended)

uv provides faster dependency resolution and installation:

git clone https://github.com/youruser/reverse-shell-killer.git
cd reverse-shell-killer
uv pip install -r requirements.txt
python reverse_shell_killer.py

Building a Standalone Binary

For deployment in production environments, building a standalone binary is recommended:

git clone https://github.com/youruser/reverse-shell-killer.git
cd reverse-shell-killer
pip install pyinstaller
pyinstaller --onefile reverse_shell_killer.py

The binary will be available in the dist/ directory.

Setting Up Environment Variables

For email notifications and LLM-based detection, create a .env file in the project root:

GEMINI_API_KEY=your_gemini_api_key_here
EMAIL_USER=your_gmail_address@gmail.com
EMAIL_PASS=your_gmail_app_password
EMAIL_RECIPIENT=security_team@yourcompany.com

Important security note: Use Gmail App Passwords, not your regular password.

Documentation

Basic Usage

python reverse_shell_killer.py

This runs the detector with default settings:

  • 10-second scan interval
  • Logs to ./revlogs.log
  • No LLM analysis
  • No metrics collection
  • Will terminate detected reverse shells

Advanced Usage Examples

# Run with LLM analysis and custom interval
python reverse_shell_killer.py --use-llm --interval 5

# Run in dry-run mode (detection only, no termination)
python reverse_shell_killer.py --dry-run

# Enable Prometheus metrics
python reverse_shell_killer.py --metrics

# Specify custom log file
python reverse_shell_killer.py --logfile /var/log/reverse-shell-killer.log

Running as a System Service

Using the Installation Script (Recommended):

sudo ./install_service.sh

The script will:

  • Copy the binary to /usr/local/bin/
  • Set up a systemd service
  • Configure environment variables
  • Start the service automatically

Detection Mechanisms

Reverse Shell Killer employs multiple detection strategies to identify malicious reverse shells:

Pattern-Based Detection

The tool maintains an extensive library of regular expressions that match known reverse shell patterns, including:

  • Bash-based reverse shells (bash -i, /dev/tcp)
  • Netcat variants (nc, netcat, ncat)
  • Python reverse shells (socket connections, pty spawning)
  • Perl, Ruby, PHP, and PowerShell reverse shell patterns
  • Obfuscated and encoded payloads

AI-Powered Detection

When enabled, suspicious processes are analyzed using Google's Gemini API. The LLM examines command-line arguments and process context to identify novel or obfuscated reverse shell techniques, providing a confidence score and explanation for each detection.

False Positive Handling

Reverse Shell Killer is designed to minimize false positives by combining multiple detection signals and allowing for user-defined whitelists. Detected processes can be reviewed in dry-run mode before enabling automatic termination.

Logging & Forensics

All detection and termination events are logged with timestamps, process details, and detection rationale. Logs are rotated automatically to prevent disk space issues.

Frequently Asked Questions

Does Reverse Shell Killer support Windows or Mac?
Currently, Reverse Shell Killer is designed for Linux systems. Windows and Mac support may be considered in future releases.
Is my data sent to Google when using LLM detection?
Only suspicious process command lines are sent to Google's Gemini API for analysis. No sensitive files or unrelated data are transmitted.
How can I reduce false positives?
Use dry-run mode to review detections before enabling automatic termination. You can also add trusted processes to the whitelist.
Can I contribute to the project?
Absolutely! Contributions, bug reports, and feature requests are welcome on our GitHub repository.
